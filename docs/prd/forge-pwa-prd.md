# Forge PWA 产品需求文档（WebApp 版）

## 1. 项目概述

### 项目名称

Forge PWA

### 项目定位

Forge PWA 是 “Forge” 训练工具的 WebApp（Progressive Web App）版本，目标是让用户在浏览器中获得接近原生 App 的训练体验。

核心能力延续原版 MVP：

- 制定与管理训练计划
- 快速开始训练并记录结果
- 查看历史与统计，跟踪长期成长

### 目标用户

- 有固定健身习惯、希望随时随地训练的人
- 经常使用手机浏览器、偏好轻量入口的人
- 在弱网/离线环境也需要继续使用记录功能的人

## 2. PWA 目标（相对原版 MVP 的增强）

| 目标 | 说明 |
|---|---|
| 可安装 | 用户可将 Forge PWA 添加到桌面/主屏，像 App 一样启动 |
| 快速启动 | 首页加载时间降低、首次启动可预缓存关键界面 |
| 离线可用 | 无网络时仍可浏览今日训练、记录动作，并在恢复网络后同步 |
| 实时提醒 | 支持训练提醒通知（训练前/倒计时到达） |
| 多端一致 | Web 与移动端浏览器行为一致，支持横竖屏训练场景 |

## 3. 范围与原则

### MVP 范围（继续沿用）

- 创建、编辑、删除训练计划
- 动作管理（次数型、时间型）
- 开始训练、记录结果
- 查看历史训练、查看基础统计

### PWA 增强范围（本版本新增）

- Web App Manifest 与安装体验
- Service Worker 与资源缓存（App Shell + 训练数据离线）
- 网络状态提示与离线记录队列
- 可选推送提醒（训练开始提醒/计时提醒）
- 首屏更新提示与应用版本更新策略
- 桌面端快捷入口（含“添加到主屏”流程兼容性提示）

## 4. 需求说明

### 4.1 首页 Dashboard

保持原版目标：打开即见今日训练状态。  
PWA 补充：

- 首屏优先展示“今日训练、完成进度、最近训练摘要”
- 在无网状态下展示最近一次缓存的数据
- 支持“离线标识”与“待同步队列数”

### 4.2 训练计划管理

保持原版功能：创建、编辑、删除、调整训练计划与动作。  
PWA 补充：

- 计划列表采用分页/懒加载，避免大列表首次阻塞
- 本地优先编辑，在线同步成功后显示“已同步/待同步”状态
- 支持从离线编辑恢复到在线环境

### 4.3 训练动作管理

- 保留次数型与时间型两类目标类型
- 每条动作支持本地草稿与历史单位
- 离线环境下可新增/编辑动作，恢复网络后批量提交

### 4.4 训练执行

- 次数型：输入目标与实际值，支持“完成”单击提交
- 时间型：内置计时器、到达目标自动提示、提前结束与超时继续
- PWA 交互补充：
  - 计时器在前后台切换时保持可恢复
  - 关键交互按钮支持大触控区域（移动端）
  - 持久化当前训练状态，防止刷新/重启导致上下文丢失

### 4.5 训练记录与统计

- 历史记录、详情页保持原样
- PWA 模式新增：
  - 离线记录在本地先入库
  - 自动重试同步失败的记录（支持指数退避）
  - 统计页应支持“当前缓存范围内统计”和“服务端最新统计”切换

## 5. 非功能性需求（WebApp）

### 5.1 可用性与体验

- 启动速度：添加 PWA 图标、首屏展示时间可接受，首次打开后再次打开明显变快
- 操作路径尽量短（训练开始≤3步）
- 重点场景支持单手操作，按钮和输入区域符合移动端可点范围

### 5.2 离线与数据一致性

- 所有“记录动作结果”接口必须具备幂等保护
- 本地队列中的记录具备冲突检测（同一训练会话重复提交需去重）
- 网络恢复后优先同步优先级：新增训练记录 > 计划变更 > 统计缓存

### 5.3 可维护性

- 缓存策略和数据版本要可配置
- 关键接口与本地存储变更需要更新兼容方案（Schema 变更时具备迁移逻辑）

## 6. 技术实现要点（建议）

| 模块 | 建议实现 | 说明 |
|---|---|---|
| 安装能力 | Web App Manifest + 安装提示 | 提供名称、图标、颜色、启动画面 |
| 缓存 | Service Worker + Cache API | 离线加载 App Shell、关键页面与静态资源 |
| 数据 | 本地存储（IndexedDB）+ 后台同步 | 训练结果先写入本地队列，再同步 |
| 网络状态 | `navigator.onLine` + 自定义重试策略 | 遇离线自动显示提示和待同步项 |
| 通知 | Notification API + 定时策略 | 用于训练提醒和时间型完成提示 |
| 更新机制 | Service Worker 版本号 + 更新横幅 | 新版本安装后提示刷新 |

## 7. 交付标准（验收）

### 功能验收

1. 用户可在支持的浏览器中安装 Forge PWA
2. 无网下可完成“查看今日训练/记录动作”
3. 恢复网络后可自动同步待同步记录
4. 时间型动作达到目标时能触发本地提醒
5. 推送功能可通过系统设置启停

### 性能验收

- 首次可用时间不超过 3 秒（在 4G/Wi-Fi 正常网络下）
- 无网启动可在 2 秒内打开最近一次首页渲染
- 训练记录提交在恢复网络 10 秒内完成一次同步（小规模队列场景）

## 8. 实施优先级（建议）

| 阶段 | 目标 | 内容 |
|---|---|---|
| P0（首发） | 训练闭环可用 | Dashboard、计划、执行、记录、历史、统计 |
| P1（PWA 基础） | 可安装+离线 | Manifest、Service Worker、基本缓存、离线记录队列 |
| P2（PWA 完善） | 体验与通知 | 推送提醒、版本更新策略、离线同步重试与状态展示 |
| P3（增强） | 稳定与优化 | 缓存命中率优化、离线冲突提示、性能再优化 |

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| iOS Safari 对部分 PWA 功能限制 | 通知/安装体验不一致 | 提供功能可见状态说明，优先保证核心流程可用 |
| 离线数据重复提交 | 数据污染 | 使用幂等 ID、会话去重策略 |
| 缓存更新策略不当 | 资源旧版本残留 | 明确缓存版本、激活时清理旧缓存并提示刷新 |
| 用户关闭通知权限 | 提醒链路失效 | 提供替代提醒（本地状态与时间提示） |

## 10. 里程碑建议

| 里程碑 | 目标交付 | 预计交付项 |
|---|---|---|
| M1 | MVP Web 功能闭环 | 核心训练管理与执行功能 |

| M3 | PWA Release | 提醒、更新策略、离线队列补齐、兼容性优化 |
