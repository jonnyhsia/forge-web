## 1. 执行原则

| 原则 | 约束 |
|---|---|
| 行为来源 | PRD 决定业务行为与验收标准 |
| 视觉来源 | Figma Make 决定页面结构、视觉语言和主交互方式 |
| 冲突处理 | PRD 与设计冲突时，先在 T00 形成决策，不由开发人员自行推断 |
| 串行执行 | T(N) 验收通过并冻结产出后，才能开始 T(N+1) |
| 独立交付 | 每项任务使用独立分支或 PR，只包含该任务范围内的改动 |
| 禁止隐式返工 | 后续任务发现前序契约错误时，先建立独立修正任务，不混入当前任务 |
| 原型使用 | Figma Make 是高保真原型，不直接复制其单文件架构、模拟数据或未使用组件库 |
| 验证约束 | 测试、构建、编译命令必须在获得用户许可后才能运行 |

## 2. MVP 范围

### 2.1 纳入范围

| 领域 | MVP 内容 |
|---|---|
| Dashboard | 周日历、每日训练、休息日、进度、最近摘要、开始/继续训练 |
| 计划管理 | 分页列表、创建、编辑、删除、训练日、动作排序 |
| 动作管理 | 次数型、时间型、目标组数、重量、次数、时长、休息时间 |
| 训练执行 | 会话恢复、次数记录、计时、暂停、提前结束、超时继续、完成总结 |
| 记录与统计 | 历史分页、详情快照、训练频率、连续训练、PR、训练量 |
| 本地优先 | IndexedDB、离线读写、同步队列、幂等、去重、重试、冲突状态 |
| PWA | 安装、App Shell 缓存、离线启动、显式更新提示、缓存版本管理 |
| 设置与通知 | kg/lbs、训练提醒、休息提醒、通知权限和站内降级提醒 |
| 适配 | 手机、横屏、桌面基础布局、安全区、软键盘、触控和无障碍 |

### 2.2 不纳入范围

| 项目 | 说明 |
|---|---|
| 意见反馈 | Figma 中为占位入口，PRD 未定义 MVP 行为 |
| 数据导出 | Figma 中为占位入口，PRD 未定义格式与范围 |
| P3 缓存优化 | 缓存命中率专项优化、复杂冲突合并和深度性能调优延后 |
| 完整桌面信息架构 | MVP 只保证桌面可用，不重新设计桌面专属产品结构 |

### 2.3 范围解释

PRD 将通知和完善的同步重试列为 P2，但功能验收又要求本地提醒和恢复网络后的自动同步。本执行文档采用严格验收口径，将这两项纳入 MVP。若需缩减范围，必须在 T00 明确调整发布门槛。

## 3. 设计与模型缺口

| 缺口 | 当前情况 | 必须在何处解决 |
|---|---|---|
| 重量与单位 | Figma 使用重量、kg/lbs、PR 和训练量；当前领域模型未覆盖重量结果 | T00、T02 |
| 每日训练日程 | Figma 支持每日多场训练与训练类别；当前只有计划星期安排 | T00、T02 |
| 计划完整管理 | Figma 的编辑计划无行为，缺删除、排序和同步反馈 | T00、T05 |
| 训练生命周期 | Figma 固定从特定动作开始，缺恢复、结束确认和完成总结 | T00、T06–T09 |
| 计时边界 | 缺提前结束确认、超时继续、前后台恢复和系统提醒 | T00、T08、T12 |
| 统计来源 | 缺缓存范围、服务端最新、更新时间和离线降级 | T00、T10 |
| PWA 状态设计 | 缺安装、离线、待同步、冲突、更新和权限状态 | T00、T11–T14 |
| 多端布局 | Figma 桌面仅展示固定手机框，不是响应式布局 | T00、T15 |
| 离线字体 | Figma 使用 Google Fonts 远程资源 | T04 |

## 5. 任务执行明细

### T00 冻结产品契约

| 项目 | 内容 |
|---|---|
| 目标 | 在开发 UI 或升级 Schema 前消除 PRD、Figma 与当前模型之间的歧义 |
| 输入 | PRD、Figma Make 原型、当前 `domain/data/store/router` |
| 工作范围 | 定义重量与单位语义、每日多场训练、训练类别、统计口径、同步服务边界、目标浏览器矩阵 |
| 产出 | MVP 范围决策、路由表、实体关系、状态矩阵、交互矩阵、API/Transport 契约、验收矩阵 |
| 验收 | Figma 中每个可见入口都有明确行为；计划、会话、离线、同步、通知和更新状态无未决项 |
| 不包含 | 代码、数据库迁移、页面实现 |

必须冻结的状态：

| 对象 | 状态 |
|---|---|
| 计划 | 有效、归档、待同步、同步失败、冲突、已删除 |
| 训练会话 | 草稿、进行中、暂停、完成、取消 |
| 日程 | 休息、计划中、进行中、部分完成、已完成 |
| 同步项 | 待同步、处理中、失败、冲突、已同步 |
| 网络 | 在线、离线、恢复中 |
| 通知 | 未请求、已允许、已拒绝、不支持 |
| PWA | 可安装、已安装、有更新、离线可用 |

### T01 建立质量基线

| 项目 | 内容 |
|---|---|
| 目标 | 为训练状态机、计时、IndexedDB 和迁移提供可重复验证环境 |
| 输入 | T00 冻结的数据与行为契约 |
| 工作范围 | 测试目录约定、Figma 示例 Fixture、Fake IndexedDB、可控时钟、可控 UUID、网络状态替身 |
| 产出 | 测试基础配置、共享 Fixture、测试数据构造器、验证命令说明 |
| 验收 | 后续任务可独立构造计划、会话、历史、统计和同步队列；时间与 ID 结果确定 |
| 不包含 | 具体业务功能测试、UI 实现 |
| 执行提醒 | 安装测试依赖或运行测试命令前必须取得用户许可 |

### T02 升级领域模型与数据库 Schema

| 项目 | 内容 |
|---|---|
| 目标 | 让正式模型覆盖 Figma 和 PRD 中的训练、重量、日程与统计语义 |
| 输入 | T00 模型决策、T01 Fixture |
| 工作范围 | 重量目标/结果、单位快照、训练日程、训练类别、PR、训练量、设置字段、Schema v1→v2 迁移 |
| 产出 | Domain v2、数据库 v2 定义、迁移逻辑、默认值策略、迁移验证用例 |
| 验收 | v1 数据可升级并读取；新字段有明确默认值；历史记录保存单位和动作快照 |
| 不包含 | 页面、具体 Repository 用例、同步 Transport |

### T03 重构数据管理边界

| 项目 | 内容 |
|---|---|
| 目标 | 避免单一 Store 和全量 `refresh()` 随功能增长形成性能与耦合问题 |
| 输入 | T02 Domain v2 与 Schema v2 |
| 工作范围 | 按 plans/workouts/history/statistics/settings 拆分 slice；分页查询；加载、错误和游标状态；独立用例接口 |
| 产出 | 功能级数据 API、分页 Repository、Store slices、统一错误类型 |
| 验收 | 任一模块可独立读取和更新；计划与历史不会全量加载；写入只刷新受影响数据 |
| 不包含 | 视觉 UI、真实同步 Transport |

### T04 建立视觉系统与应用壳

| 项目 | 内容 |
|---|---|
| 目标 | 将 Figma 视觉语言转换为可复用且离线可用的生产基础 |
| 输入 | T00 路由/状态矩阵、Figma `theme.css` 与页面结构 |
| 工作范围 | 颜色、字体、间距、圆角、图标；字体本地化；按钮、卡片、表单、弹层、进度组件；路由、底部导航、全屏训练壳 |
| 产出 | Design Token、通用组件、应用壳、空/加载/错误/离线状态、正式路由 |
| 验收 | 所有路由可直达、返回和刷新；底部导航状态正确；字体离线可用；页面壳支持安全区 |
| 不包含 | 任何业务数据闭环 |

正式路由基线：

| 路径 | 页面 |
|---|---|
| `/` | Dashboard |
| `/plans` | 计划列表 |
| `/plans/new` | 新建计划 |
| `/plans/:planId` | 编辑计划 |
| `/training/start` | 创建训练会话 |
| `/training/:sessionId` | 训练执行 |
| `/history` | 历史列表 |
| `/history/:sessionId` | 历史详情 |
| `/statistics` | 统计 |
| `/settings` | 设置 |

### T07 实现次数型训练

| 输入 | T00–T14 的冻结产出 |
| 工作范围 | 手机、横屏、桌面、安全区、软键盘、触控尺寸、焦点、对比度、性能、在线/离线/弱网回归 |
| 产出 | 发布候选版本、验收记录、已知限制和回滚方案 |
| 验收 | 首次可用 ≤3 秒；无网首页 ≤2 秒；恢复网络 10 秒内发起同步；关键交互满足移动端触控要求 |
| 不包含 | 新功能、P3 优化 |
| 执行提醒 | 构建、测试、编译和完整回归命令必须在获得用户许可后执行 |

## 6. 串行门禁

每个任务完成时必须提交以下信息，缺少任一项不得进入下一任务：

| 门禁项 | 要求 |
|---|---|
| 范围核对 | 实际改动仅属于当前任务 |
| 产出清单 | 文件、接口、页面或文档与任务定义一致 |
| 验收记录 | 逐条记录通过、未通过或未执行及原因 |
| 设计核对 | UI 任务说明与 Figma 的差异及理由 |
| 数据兼容 | 涉及数据时说明 Schema、默认值和回滚影响 |
| 风险更新 | 新风险进入风险表，不隐式留给后续任务 |
| 用户许可 | 运行构建、测试或编译前记录已获得许可 |

## 7. 发布门槛

| 门槛 | 必须完成 | 可用于 |
|---|---|---|
| Foundation Ready | T00–T04 | 开始功能开发 |
| Core Feature Complete | T00–T11 | 内部核心流程演示，不称为 PWA MVP |
| PWA MVP Complete | T00–T14 | PWA Beta 候选 |
| Release Candidate | T00–T15 | 按 PRD 进入发布决策 |

## 8. 风险与阻塞项

| 风险 | 影响 | 处理任务 |
|---|---|---|
| 同步服务端接口未定义 | 无法完成真实多端同步验收 | T00、T13 |
| 重量/单位语义未冻结 | 会引发 Schema 二次迁移 | T00、T02 |
| 每日多场训练语义不清 | Dashboard 与计划模型可能重复 | T00、T02、T11 |
| Figma 缺少 PWA 状态画面 | 离线、同步、更新体验可能不一致 | T00、T04、T11、T14 |
| Figma 桌面仅为手机框 | 无法直接作为桌面适配依据 | T00、T15 |
| 远程字体影响离线体验 | 无网时品牌字体失效 | T04、T14 |
| Store 全量刷新 | 数据增长后列表性能下降 | T03 |
| 计时依赖运行中定时器 | 切后台后时间可能漂移 | T08 |
| 未经许可不能运行验证命令 | 验收状态可能停留在未执行 | 每个任务门禁 |

## 9. 任务执行模板

后续领取任一任务时，应使用以下格式：

| 项目 | 填写内容 |
|---|---|
| 任务编号 | Txx |
| 目标 | 本任务唯一业务结果 |
| 输入 | 已冻结的前序契约与接口 |
| 范围 | 本任务允许修改的模块 |
| 非范围 | 明确禁止顺带实现的内容 |
| 设计依据 | Figma 页面、组件或状态 |
| 数据影响 | Domain、Schema、迁移和同步影响 |
| 验收标准 | 可逐条确认的行为结果 |
| 验证计划 | 静态检查、测试、构建或手工场景；命令执行需先获许可 |
| 风险 | 当前任务可能阻塞下一任务的事项 |
